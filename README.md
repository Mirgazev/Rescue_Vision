# Rescue Vision

Подсистема технического зрения для мобильного робота, выполняющего поисково-спасательные операции в условиях ухудшенной видимости: низкая освещённость, туман, дым и осадки.

Выпускная квалификационная работа бакалавра по направлению 15.03.06 «Мехатроника и робототехника», профиль «Автономные роботы», РТУ МИРЭА, 2026.

## Главные результаты

| Метрика | Значение | Требование ТЗ |
|---|---:|---:|
| mAP@0.5 на ExDark | **0,8905** | ≥ 0,77 |
| Точность AUTO-классификатора | **90,5 %** | ≥ 80 % |
| Производительность | **20,8–62 FPS** | ≥ 20 FPS |
| mAP на ручной GT-разметке видео МЧС | **0,82** | — |

## Архитектура

```text
[Захват кадра]
      ↓
[Анализатор сцены]
      ↓
[Классификатор режима]
      ↓
[Модуль улучшения: NIGHT / FOG / SMOKE / RAIN / OFF]
      ↓
[YOLO11-pose v4]
      ↓
[ByteTrack]
      ↓
[Публикация результатов в ROS2]
```

## Режимы обработки

| Режим | Алгоритм | Назначение |
|---|---|---|
| NIGHT | CLAHE + гамма-коррекция | Низкая освещённость |
| FOG | Dark Channel Prior | Туман |
| SMOKE | Single-Scale Retinex | Дым |
| RAIN | Медианный и билатеральный фильтр | Осадки |
| OFF | Без обработки | Нормальная видимость |
| AUTO | Автоматический выбор | Выбор режима по признакам кадра |

## Установка

```bash
git clone https://github.com/Mirgazev/Rescue_Vision.git
cd Rescue_Vision
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

## Использование

```bash
python -m rescue_vision.rescue_vision.main --source path/to/video.mp4 --enhance auto
```

ROS2 RealSense:

```bash
ros2 launch rescue_vision pipeline_realsense.launch.py
```

ROS2 rosbag:

```bash
ros2 launch rescue_vision pipeline_rosbag.launch.py path:=/path/to/rosbag
```

## Главный научный результат

Визуальное улучшение кадра не эквивалентно улучшению входа для дообученного детектора. После дообучения модели принудительное применение режимов NIGHT, FOG и SMOKE может снижать mAP на чистой сцене. Поэтому режим OFF является полноценным рабочим режимом, а AUTO используется как консервативный механизм включения улучшения только при необходимости.

## Что не входит в репозиторий

В репозиторий не загружаются веса моделей, датасеты, видеофайлы МЧС и персональные данные. Эти файлы исключены через `.gitignore`.
