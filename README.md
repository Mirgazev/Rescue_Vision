# Rescue Vision

Подсистема технического зрения для мобильного робота, выполняющего поисково-спасательные операции в условиях ухудшенной видимости (низкая освещённость, туман, дым, осадки).

Выпускная квалификационная работа бакалавра по направлению 15.03.06 «Мехатроника и робототехника», профиль «Автономные роботы», РТУ МИРЭА, 2026.

## Архитектура

Подсистема представляет собой последовательный конвейер из 6 функциональных блоков:

```
[Захват кадра] → [Анализатор сцены] → [Классификатор режима]
                                            ↓
[ByteTrack] ← [YOLO11-pose v4] ← [Модуль улучшения: DCP / Retinex / CLAHE]
     ↓
[Публикация в ROS2]
```

5 режимов улучшения видеопотока:

| Режим | Алгоритм | Назначение |
|---|---|---|
| NIGHT | CLAHE + гамма-коррекция | Низкая освещённость |
| FOG | Dark Channel Prior | Туман |
| SMOKE | Single-Scale Retinex | Дым |
| RAIN | Медианный + билатеральный фильтр | Осадки |
| OFF | Без обработки (защитный режим) | Нормальная видимость |
| AUTO | Автоматический выбор | По 5 признакам кадра |

## Установка

### Требования

- Ubuntu 20.04 (для ROS2 Foxy) или Ubuntu 24.04
- Python 3.10+
- NVIDIA GPU с CUDA 12.1+ (минимум RTX 2060, 6 ГБ VRAM)
- ROS2 Foxy (опционально, для интеграционных сценариев)

### Через pip

```bash
git clone https://github.com/<your-username>/rescue_vision_bachelor.git
cd rescue_vision_bachelor
pip install -r requirements.txt
```

### Сборка ROS2-пакета

```bash
cd <your_ros2_workspace>/src
ln -s <path_to_rescue_vision_bachelor>/rescue_vision .
ln -s <path_to_rescue_vision_bachelor>/rescue_vision_msgs .
cd ..
colcon build --packages-select rescue_vision rescue_vision_msgs
source install/setup.bash
```

## Использование

### Standalone-режим (без ROS2)

```bash
python rescue_vision/main.py --source path/to/video.mp4 --enhance auto
```

### С камерой RealSense через ROS2

```bash
ros2 launch rescue_vision pipeline_realsense.launch.py
```

### Воспроизведение rosbag-записи

```bash
ros2 launch rescue_vision pipeline_rosbag.launch.py path:=/path/to/rosbag
```

### Изменение параметров на лету

```bash
ros2 param set /rescue_vision_node enhance_mode night
ros2 param set /rescue_vision_node conf_threshold 0.25
ros2 param set /rescue_vision_node imgsz 640
```

## Главный научный результат

**Визуальное улучшение кадра не эквивалентно улучшению входа для дообученного детектора.**

После дообучения модели на смешанной выборке ExDark + RESIDE + COCO принудительное применение режимов NIGHT, FOG, SMOKE снижает mAP в среднем на 3-7 п.п. на чистой сцене. Поэтому в архитектуре системы режим OFF — это полноценный рабочий режим, а AUTO — консервативный механизм с пороговым включением.

Подробное обоснование — в [пояснительной записке ВКР](docs/thesis.pdf).

|---|---|
